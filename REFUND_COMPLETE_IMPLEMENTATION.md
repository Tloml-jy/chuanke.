#### 审核退款功能
**操作流程**：
1. 点击"审核退款"按钮
2. 弹出确认框（通过/驳回）
3. 选择通过 → 直接审核通过
4. 选择驳回 → 弹出输入框填写驳回原因
5. 提交后刷新列表

**代码实现**：
```javascript
onReviewRefundTap(e) {
  // 1. 显示确认框
  wx.showModal({
    title: '审核退款',
    content: '是否通过该订单的退款申请？',
    confirmText: '通过',
    cancelText: '驳回',
    success: async (res) => {
      if (res.confirm) {
        // 审核通过
        await call('order', {
          action: 'reviewRefund',
          orderId: id,
          merchantId: merchantId,
          approved: true
        })
      } else {
        // 审核驳回，需要填写原因
        const reason = await askRejectReason()
        await call('order', {
          action: 'reviewRefund',
          orderId: id,
          merchantId: merchantId,
          approved: false,
          rejectReason: reason
        })
      }
    }
  })
}
```

## 三、状态机设计

### 3.1 完整状态流转

```
待支付(unpaid)
    ↓ 用户支付
已支付(paid)
    ↓ 用户申请退款
已支付(paid) + 退款待审核(pending_review)
    ↓ 商户审核通过
已支付(paid) + 退款已审核(approved)
    ↓ 商户执行退款
退款处理中(refund_processing)
    ↓ 微信退款成功
已退款(refunded)

或者：
退款待审核(pending_review)
    ↓ 商户审核驳回
已驳回(rejected) → 回到已支付状态
```

### 3.2 部分退款流程

```
已支付(paid)
    ↓ 部分退款成功
部分退款(partial_refunded)
    ↓ 再次申请退款
部分退款(partial_refunded) + 退款待审核(pending_review)
    ↓ ...
```
